# Explain Code

Explain Code stores learning episodes generated by Codex and Claude Code.

## Quick Helm Install

After pushing the frontend and backend images to Docker Hub, install the app on k3s from the packaged Helm chart.

### Direct Chart URL

```bash
helm upgrade --install explain-code \
  https://raw.githubusercontent.com/dong7314/explain-code/master/charts/explain-code-0.1.0.tgz \
  -n explain-code \
  --create-namespace \
  -f https://raw.githubusercontent.com/dong7314/explain-code/master/deploy/helm/explain-code/examples/values-traefik-cert-manager.yaml \
  --set global.publicHost=your-domain.example.com
```

### Helm Repo

```bash
helm repo add explain-code https://raw.githubusercontent.com/dong7314/explain-code/master/charts
helm repo update

helm upgrade --install explain-code explain-code/explain-code \
  -n explain-code \
  --create-namespace \
  -f https://raw.githubusercontent.com/dong7314/explain-code/master/deploy/helm/explain-code/examples/values-traefik-cert-manager.yaml \
  --set global.publicHost=your-domain.example.com
```

`global.publicHost` is the only domain value you need for the normal same-domain setup. The chart uses it for ingress host, TLS host, and backend CORS. The frontend keeps `apiBaseUrl: /api`, so the browser calls `https://your-domain.example.com/api`.

## Quick Install For AI Clients

### Windows PowerShell

```powershell
irm https://raw.githubusercontent.com/dong7314/explain-code/master/scripts/install-explain-code.ps1 | iex
```

### macOS / Linux

```bash
curl -fsSL https://raw.githubusercontent.com/dong7314/explain-code/master/scripts/install-explain-code.sh | bash
```

The installer copies:

- Codex skill: `~/.codex/skills/explain-code-learning`
- Shared publisher tool: `~/.explain-code/tools/explain-code-ingest`
- Claude Code plugin files: `~/.explain-code/integrations/claude-code-plugin`

Before publishing, set an API token issued from Explain Code.

```powershell
$env:EXPLAIN_CODE_API_TOKEN="expc_live_..."
```

For a local backend, the default API URL is already `http://localhost:4000/api`.

Optional overrides:

```powershell
$env:EXPLAIN_CODE_API_URL="http://localhost:4000/api"
```

Codex and Claude Code choose the learning group automatically. They list
existing groups, reuse a matching group when one exists, and create a new group
when needed.

Codex needs a restart after installing the skill.

Claude Code can be started with the installed plugin:

```powershell
powershell -ExecutionPolicy Bypass -File "$env:USERPROFILE\.explain-code\claude-explain-code.ps1"
```

Then run:

```text
/explain-code:publish-learning
```
